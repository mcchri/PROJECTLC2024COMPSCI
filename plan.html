The game

The memory game is codded into the microbit. There are four patterns to be memorized. The levels and the amount of patterns keeps incrementing until the user presses the wrong pattern for the following stored pattern sequence. Here is a flow chart diagram explaining how the game works. My analog input is the light sensor and my digital input are the buttons. My analog output is the sound coming from the microbit. My digital outputs are the LEDs.

Microbit flowcharts

The age input:

The game starts by the user entering their age by them pressing button A and then outputs the text "age" also age = 0. If the button A is pressed the age increases by 10, if button B ia pressed the age will increase by one and if pin 0 is pressed then it the user will choose the level of the game (how fast the images will appear on the LED screen).

The level input:

If button B is pressed the level increases by one, if the level is equal to 3 and the button B is pressed then the level is equal to 1. If pin 0 is pressed then record the current light level (ANALOG INPUT). Output the text "level 1", initialise the game level, iterator and the list of LEDs.

The LED output:

A random number is chosen from 1 to 4. The random number is placed within the list, the iterator goes through all the numbers in the list. If the number is 1 then output a heart symbol, if the number is 2 then output a skull symbol, if the number is 3 then output a smile symbol and if it is 4 then display the letter B. For every cheking increase the iterator by one to iterate through the list. If the iterator is equal to the last number then go to the buton logic, if the right buttons are pressed for the sequence then add another random number to the list. If the button logic is false then end game.

The button logic:

Initialise an iterator. If button B pressed then button_input=1, if pin 0 pressed then button_input=2, if pin 1 pressed then button_input3, if pin 2 pressed then button_input=4. If the input equals to the first item to the list and if the iterator equals to the sizse of the list minus 1 then iterator=0, output an analog output as a happy noise and output all the symbols from the list. If the iterator does not equal to the list then add 1 to it and start the conditioning again until the wrong button is pressed. Output the score if the wrong button is pressed and output the serial output of light sensitivity. Output a sad noise to confirm the loss and end the game.

The architecture of my design and the technologies used:

The microbit recieves the code from the PC in order to execute the game that I made. The light level and the level of the game is send back to the PC. I used pyhton to send my data to the firebase. I retrieved the filtered data with javascript and made a graph using the graph module.

User Requirements:

This game will increase the attention span of the person by making the person memorize the patterns shown on the LED. Attention span is directly linked to the wellbeing of a person because they will be more productive and stimulated. The light levels of the room can affect how well the patterns are memorized. The end user wants to increase their attention span then this game will meet their requirements.
