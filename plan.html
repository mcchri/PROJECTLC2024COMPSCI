Plan and design

The game

The memory game is coded into the microbit, featuring four patterns for memorization. Levels and pattern count increase until the user makes an incorrect sequence input. The game's functionality is detailed in a flow chart. Inputs include the light sensor (analog) and buttons (digital), while sound (analog) and LEDs (digital) serve as outputs.

Microbit flowcharts

The age input:

The game begins with the user inputting their age via button A, initializing age to 0. Pressing button A increments age by 10, button B increases it by one, and pin 0 selects the game level (speed of LED screen images).

The level input:

If button B is pressed the level increases by one, if the level is equal to 3 and the button B is pressed then the level is equal to 1. If pin 0 is pressed then record the current light level (ANALOG INPUT). Output the text "level 1", initialise the game level, iterator and the list of LEDs.

The LED output:

A random number (1-4) is selected and added to a list. Each number in the list corresponds to a symbol: 1 for heart, 2 for skull, 3 for smile, and 4 for B. The iterator traverses the list, incrementing with each check. If the iterator reaches the last number, the game proceeds to button logic. Correct button presses prompt adding another random number; otherwise, the game ends.

The button logic:

Initialize an iterator. Assign button_input values based on button or pin presses. If input matches the first item in the list and iterator equals list size minus 1, reset iterator, output a happy noise, and display all symbols. If iterator doesn't match the list, increment it until a wrong button is pressed. Output score and light sensitivity if wrong, then end game with a sad noise.

The architecture of my design and the technologies used:

The microbit recieves the code from the PC in order to execute the game that I made. The light level and the level of the game is send back to the PC. I used pyhton to send my data to the firebase. I retrieved the filtered data with javascript and made a graph using the graph module.

User Requirements:

This game enhances attention span by requiring players to memorize LED patterns. Improved attention span boosts productivity and stimulation, crucial for overall well-being. Room light levels influence pattern memorization effectiveness. For users seeking attention span enhancement, this game aligns with their needs.
