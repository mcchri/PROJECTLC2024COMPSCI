Plan and design

I chose the Agile method on making my project because it is very flexible and suits my requirements.

Universal design concepts can be seen with the game and the website. The simple understood button symbols.

Basic Requirements:


My system is fully automated, with data sent automatically to Firebase from the microbit and being sent to javascript to automatically create a graph.


Advanced Requirements:


A dataset of wellbeing data, including user age, memory score, and light level, was created and a Python program was developed to compare memory scores across age groups. Graphically presenting with a graph.

Model

My model is a feedback model, it gives back feedback for the user and helps the user to understand what state of mental well-being they are in.

My 2 what-if questions are:
  1. What if the age affects how well the player can memorize the sequence? The three validated parameters are user reading books, user under 18, and average age of total game players, which correlate with mental age.

  2. What if the light intensity affects how well the player can memorize the sequence?The three validated parameters, location, sufficient light source, and average light intensity, are crucial in understanding how the environment impacts thinking and well-being.

The game

The microbit memory game uses four patterns for memorization, with levels and pattern count increasing until incorrect input is made. Inputs include light sensor (analog), buttons (digital), sound(analog), and LEDs(digital).

Microbit flowcharts

The age input:

The user inputs their age, which is then adjusted by pressing buttons A, B, +10 and +1 respectively.

The level input:

Press button B to increase level, pin 0 to record light level (ANALOG INPUT), output "level 1", initialize game level, iterator, and LED list.

The LED output:

A random is added to a list representing symbols, and the iterator iterates through the list, incrementing each check until the last number is reached.

The button logic:

Initialize an iterator, assign button input values, reset if inputs matches first item, increment if wrong, output score and light sensitivity, end game with sad noise.

The architecture of my design and the technologies used:

The microbit recieves the code from the PC in order to execute the game that I made. The light level and the level of the game is send back to the PC. I used pyhton to send my data to the firebase. I retrieved the filtered data with javascript and made a graph using the graph module.
